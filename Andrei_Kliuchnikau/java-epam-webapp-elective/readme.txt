Тема проекта (предметная область): Система "Факультатив"

Описание проекта:
Существует перечень Курсов, за каждым из которых закреплён один Преподаватель. Студент записывается на один или несколько Курсов. По окончании обучения Преподаватель выставляет Студенту Оценку и добавляет отзыв.

Требования к проекту:
Task 4
Разработать подсистему для работы с базой данных предложенной предметной области:
1. Разработать схему базы данных в соответствии с предметной областью вашего варианта. Создать sql-скрипты создания БД, пользователя БД, создание таблиц, заполнение таблиц, удаление данных, удаление таблиц, удаление БД, обновление данных, запросы на выборку данных.
2. Информацию о предметной области хранить в БД, для доступа использовать API JDBC с использованием пула соединений, разработанного самостоятельно. В качестве СУБД используется MySQL.
3. На основе сущностей предметной области создать классы их описывающие.
4. Классы и методы должны иметь отражающую их функциональность названия и должны быть грамотно структурированы по пакетам.
5. Оформление кода должно соответствовать Java Code Convention.
6. Приложение должно поддерживать работу с кириллицей (быть многоязычной), в том числе и при хранении информации в БД.
7. Выполнить журналирование событий, то есть информацию о возникающих исключениях и событиях в системе обрабатывать с помощью Log4j 2.
8. Код должен содержать комментарии.

Техническая информация:
Файл для инициализации базы данных находится в папке sql (init.sql). Возможна инициализация и удаление БД через файлы db-init.bat и db-delete.bat (предварительно поменяв параметры подключения к БД).
В папке \sql\scripts хранятся отдельные скрипты согласно пункту 1 задания 4.
Параметры подключения к БД из приложения находятся в файле database.properties по пути \src\by\training\resources.
Собранное приложение хранится в файле \elective-archive.war
Для работы необходим JDK 11+

Функционал:
Имеется три группы пользователей (данные входа в систему ниже) - администраторы, преподаватели студенты. 
Администраторы могут создавать учётные записи и присваивать (раздавать) их преподавателям. 
Преподаватели могут присваивать (раздавать) учётные записи студентам, а также создавать новые курсы и выставлять оценки. Помимо основного функционала весь преподавательский состав имеет доступ к универсальному интерфейсу работы (могут редактировать/удалять/создавать) с любыми курсами, записями на курсы, оценками и студентами. 
Студенты: могут ТОЛЬКО просматривать курсы и записываться на интересующие их (*сроки прохождения курса устанавливается преподавателем отдельно для каждого студента после прохождения регистрации на курс). 
Удаление сущностей из системы возможно только при отсутствии у них связей (нельзя удалить из системы преподавателя, закреплённого за каким-то курсов, как и нельзя удалить студента, который проходит обучение на каком-то курсе).
Система предусматривает поиск в таблицах по определённым полям.
В личном кабинете можно сменить пароль и e-mail.
